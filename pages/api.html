API Reference

Integrate ReportDrafter into your workflows with our REST API. Upload, extract, and download reports programmatically.

API Version: v1 (Beta) · Base URL: https://api.reportdrafter.com/v1

Authentication

All API requests require a Bearer token in the Authorization header. Generate an API key from your workspace settings under Settings → API Keys.

Authorization: Bearer rd_live_xxxxxxxxxxxxxxxxxxxx

Endpoints

GET /templates

List all report templates in the current workspace. Returns template ID, name, field count, and creation date.

POST /templates

Upload a new report template (.docx or .xlsx). The system parses the template structure and detects extractable fields.

curl -X POST https://api.reportdrafter.com/v1/templates \
  -H "Authorization: Bearer rd_live_xxx" \
  -F "file=@quarterly_report_template.docx"
GET /templates/{id}

Retrieve details for a specific template, including its detected fields (name, type, position).

DELETE /templates/{id}

Delete a template and all associated reports. This action cannot be undone.

Source Documents

POST /documents

Upload a source document (.pdf, .docx, .xlsx) to be used during extraction. Returns a document ID.

GET /documents

List all uploaded source documents in the workspace.

Extraction

POST /extractions

Start an AI extraction job. Provide the template ID and one or more document IDs. Returns a job ID and status.

curl -X POST https://api.reportdrafter.com/v1/extractions \
  -H "Authorization: Bearer rd_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "template_id": "tmpl_abc123",
    "document_ids": ["doc_def456", "doc_ghi789"]
  }'
GET /extractions/{id}

Check extraction status and retrieve results. Returns field values with confidence scores once processing is complete.

Response example:

{
  "id": "ext_xyz789",
  "status": "completed",
  "fields": [
    {
      "name": "Fund Name",
      "value": "Horizon Growth Fund II",
      "confidence": 0.98,
      "locked": false
    },
    {
      "name": "NAV Date",
      "value": "2026-02-28",
      "confidence": 0.95,
      "locked": false
    }
  ]
}
PUT /extractions/{id}/fields/{field_name}

Update a specific field value or lock/unlock it. Use this after manual review to correct AI-extracted values.

Reports

POST /reports

Generate a final report from a completed extraction. The report uses approved field values and applies them to the original template layout.

GET /reports/{id}/download

Download the generated report file (.docx or .xlsx). Returns the binary file with appropriate content headers.
